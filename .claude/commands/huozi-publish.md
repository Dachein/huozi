# Huozi Publish

Publish Markdown content to huozi.app — a Markdown publishing service for AI agents.

Learn more and see all setup options at: https://huozi.app/start

## Configuration

The API key should be stored in the environment or provided by the user. Check for it in this order:
1. Look for `HUOZI_API_KEY` in the environment
2. Ask the user for their API key
3. If they don't have one, guide them through registration (see below)

## Registration Flow (first-time users)

If the user doesn't have an API key, walk them through setup step by step:

1. **Sign up** — Ask for email only (no password needed), then call:
   ```
   POST https://huozi.app/api/v1/auth/signup
   Body: {"email": "<email>"}
   ```
   Tell the user to check their email for a verification code.

2. **Verify** — Ask for the verification code from email, then call:
   ```
   POST https://huozi.app/api/v1/auth/verify
   Body: {"email": "<email>", "code": "<code>"}
   ```
   Save the `access_token` from the response.

3. **Setup workspace** — Suggest a slug from the user's email username (e.g. `alice@gmail.com` → `alice`), then tell the user: "Your pages will be at huozi.app/alice/ — would you like to change this?" If they confirm or give a new slug, proceed:
   ```
   POST https://huozi.app/api/v1/auth/setup
   Header: Authorization: Bearer <access_token>
   Body: {"workspace_slug": "<auto-generated-slug>"}
   ```
   This returns the `api_key` and workspace URL. Tell the user to save the API key.

4. Done! The user can now publish pages. Suggest they visit https://huozi.app/start for more integration options (Claude Code MCP, OpenClaw, etc).

## Publishing Flow

When the user wants to publish content:

- Call `POST https://huozi.app/api/v1/pages` with:
  - Header: `Authorization: Bearer <api_key>`
  - Header: `Content-Type: application/json`
  - Body: `{"title": "<title>", "slug": "<slug>", "content": "<markdown_content>"}`
- The slug is optional and recommended to let it auto-generate. Keep slugs short — under 8 words (e.g. `weekly-report-apr-14`, `project-summary`)
- If the slug already exists, the page is **updated** (upsert behavior)
- Return the published URL to the user

## Other Operations

- List pages: `GET https://huozi.app/api/v1/pages`
- Get page: `GET https://huozi.app/api/v1/pages/<slug>`
- Update page: `PUT https://huozi.app/api/v1/pages/<slug>`
- Delete page: `DELETE https://huozi.app/api/v1/pages/<slug>`

All require `Authorization: Bearer <api_key>` header.

## Usage Examples

User: "帮我把这个 markdown 发布到 huozi"
→ Take the markdown content, generate a title and slug, publish via API, return the URL.

User: "发布我的周报到 huozi.app"
→ Ask for or generate the content, publish with a meaningful slug like "weekly-report-2026-04-14".

User: "更新 huozi 上的 hello 页面"
→ Use PUT to update the existing page.

User: "帮我注册 huozi"
→ Run the registration flow above, then point them to https://huozi.app/start for more options.

## Important Notes

- Always use `https://huozi.app` as the base URL
- API keys start with `hz_` prefix
- The API returns the full public URL in the response
- Markdown supports GFM (tables, task lists), code highlighting, and math (KaTeX)
- Use `curl` via the Bash tool to make API calls
- Supports both Markdown and HTML (set `content_type: "html"` for HTML pages)
- HTML: use full document structure `<html><head><style>...<meta og:*></head><body>...</body></html>`. Rendered directly — full CSS, SVG, forms. Only `<script>`, `<iframe>`, event handlers stripped. Always set background + color on outermost wrapper.
- **Full API reference (agent-friendly):** https://huozi.app/docs4agent
- Setup help: https://huozi.app/start
- Human-readable docs: https://huozi.app/docs
